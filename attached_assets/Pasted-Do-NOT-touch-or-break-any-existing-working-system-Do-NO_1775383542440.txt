We now need to build the human-in-the-loop learning workflow for AskiMate.

GOAL:
1. AI should answer when it can.
2. If AI is unsure or the topic is sensitive, human should be able to take over.
3. When human replies, there must be an option to approve that reply for the knowledge base.
4. Approved human replies should be stored for future ingestion into the KB.
5. Later, AI should be able to answer similar questions more confidently using those approved entries.

Build this in the safest and smallest possible way.

TASKS:

STEP 1 — Human handoff visibility

- user question
- AI answer
- reviewLevel

STEP 2 — Human reply approval option
When the human/mentor sends a reply, add a checkbox or toggle:

"Approve this reply for Knowledge Base"

STEP 3 — Store approved replies
If the checkbox is selected when the human sends the reply, store a new structured record in a safe place.

Create a lightweight storage mechanism for now.
Use either:
- a JSON file like ai/approved_kb_entries.json
or
- a minimal DB table only if the app already has a simple place for this

Preferred structure for each approved record:
{
  "id": "...",
  "sourceQuestion": "...",
  "humanAnswer": "...",
  "normalizedQuestion": "...",
  "domain": "...",
  "risk_level": "...",
  "approvedForKb": true,
  "approvedBy": "...",
  "approvedAt": "...",
  "status": "pending_ingest"
}

STEP 4 — Domain and risk assignment
When saving approved entries:
- infer domain from the AI top source if available
- infer risk_level from the current conversation/reviewLevel if available
