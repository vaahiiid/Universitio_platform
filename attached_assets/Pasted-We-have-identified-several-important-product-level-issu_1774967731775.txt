ISSUE 1 — CHAT PAGE HAS SMALL REPEATED JUMPING / FLICKER
==================================================

Problem:
While using chat (both admin and user side), the chat page slightly jumps/flickers every few moments.

This likely comes from polling / re-render / scroll behavior.

Required fix:
- stop the chat UI from visually jumping
- keep the page stable while polling/refetching
- do not reset the whole message area every few seconds
- do not force scroll/jump unless there is a real new message and the user is already near the bottom
- preserve input focus while polling
- avoid unnecessary full component re-renders

Please check:
- polling interval behavior
- state replacement vs state merge
- auto-scroll logic
- key usage in lists
- full component remounting
- focus loss / restore behavior

Expected result:
- chat stays visually stable
- no small repeated jumping
- no annoying flicker on either admin or user side

ISSUE 2 — REAL NOTIFICATIONS / UNREAD BADGES ARE STILL NOT WORKING PROPERLY
==================================================

Problem:
Chats now arrive, but there is still no proper notification:
- no sound
- no reliable unread number badge next to Messages
- not for admin
- not for user

We need proper in-app notification behavior while the user/admin is inside their dashboard/admin area but not necessarily inside the active chat thread.

Required behavior:
1. If the user receives a mentor reply while on another page/tab inside their account:
   - show unread badge count next to Messages
   - optionally play a subtle notification sound

2. If admin receives a user message while inside admin but not inside that conversation:
   - show unread badge count in the relevant admin navigation/section
   - optionally play a subtle notification sound

3. Unread count must update automatically without full page refresh

4. Opening the conversation should reset the unread count correctly

Important:
- unread count must reflect unread messages only
- not total conversation count
- the notification state should be visible outside the chat view too
- badge placement must be obvious and useful
